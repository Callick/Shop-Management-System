# Shop-Management-System
🛠 Overview 
[Click here for an overview](https://rawcdn.githack.com/Callick/Shop-Management-System/a1a2b13d5a1b017fee96e01202545f44eb9a511f/Run_me_for_project_overview.html)
<br>The Shop Management System is a web-based application designed to replace traditional paper-based repair tracking in service shops. It provides a secure, role-based interface for managing customer repair records, handling deletion workflows, and delivering notifications. Built on the Laravel framework with MySQL/PostgreSQL backend, the system ensures data integrity, operational efficiency, and an intuitive user experience.</br>
## ✅ Key Features
**Role-Based Access Control (RBAC)**<BR>
Supports two dedicated roles, Admin and Assistant-Admin. Ensuring secure and permissioned operations.

**Repair Record Management**<BR>
Comprehensive form to capture customer device details, payment info, repair issues, and etc.

**Deletion Approval Workflow**<BR>
Implements a controlled request-and-review mechanism for record deletions to ensure accountability.

**Notification System**<BR>
Sends system alerts (like deletion request statuses) to relevant users for real-time updates.

**Relational Database Model**<BR>
A normalized database schema defines key relationships among customers, repair_records, users, deletion_requests, and notifications.
