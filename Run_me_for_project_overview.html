Project Introduction

This system is designed to replace the existing paper-based process for tracking customer belongings, providing a secure, efficient, and reliable digital solution. It will manage repair job records, user access, and system notifications through an intuitive web interface. The primary goal is to streamline operations, reduce paperwork, and improve data accuracy for the repair shop. N.B. These will be changed on time as needed.

Key System Attributes

Role-Based Access

Strict permissions for 'Admin' and 'Assistant-Admin' roles ensure users only access the features they are authorized for.

Data Integrity

Comprehensive input validation will be implemented to ensure data consistency and protect against common web vulnerabilities.

Modern & Usable

The user interface will be intuitive, responsive, and easy to navigate, ensuring a smooth experience for all users.

User Roles & Permissions

The system features a strict Role-Based Access Control (RBAC) model with two distinct user roles. This section breaks down the specific capabilities of each role. N.B. These will be changed on time as needed.

Administrator

Full system privileges for management and oversight.

  • ✓ Full CRUD on Repair Records
  • ✓ Manage All User Accounts
  • ✓ Approve/Deny Deletion Requests
  • ✓ View System-wide Logs

Assistant-Admin

Restricted privileges for daily operational tasks.

  • ✓ Create, Read, Update Records
  • ✓ Print Repair Statements
  • ✗ Directly Delete Records
  • → Must Request Record Deletion
  • ✗ Manage User Accounts

Record Management

This is the heart of the system. All information related to a customer's repair job is captured in a single, comprehensive record. Below are the fields that make up each repair ticket. N.B. These will be changed on time as needed.

Sample Repair Record Fields

  • Customer Name: String
  • Customer Contact: String
  • Device/Item ID: String
  • Date of Receipt: Date
  • Expected Delivery: Date
  • Total Cost ($): Decimal
  • Amount Paid ($): Decimal
  • Amount Due ($): Decimal
  • Record Status: Enum
  • Stated Issues: Text
  • Comments: Text

Deletion Approval Workflow

To prevent accidental data loss, a special workflow is required when an Assistant-Admin needs to delete a record. This ensures that a senior staff member reviews and confirms every deletion. Follow the steps below to see how it works.

1

Request

Assistant-Admin initiates a 'delete' action on a record.

2

Review

System flags the record as 'Pending Deletion' and notifies all Admins.

3

Confirmation

Admin approves or denies. A final notification is sent back to the Assistant-Admin.
